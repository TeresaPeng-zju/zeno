# Zeno · 实现要点（Interview Notes）

> 本文件记录 Zeno 的核心实现与设计权衡，供面试讲解 / 内部参考。
> **不要把这些内容放进公开 README** —— 其中多处是项目的核心壁垒。

---

## 1. 决策与表达分离（核心架构）

这是整个系统最关键的设计，也是最不该对外暴露的部分。

- **决策层**：能力匹配、差距计算、下一步排序，全部由**确定性规则引擎**完成。同样的输入永远得到同样的 IDs / 分数 / 排序，与语言无关，可被离线评测稳定回归。
- **表达层**：LLM 只负责把决策结果「翻译」成自然语言解释。未配置或调用失败时回退模板文案。
- **收益**：打分结果可解释、可测试、可复现；LLM 的不确定性被隔离在表达层，不污染核心逻辑。

## 2. 能力诊断引擎

- 内置 24 个技能维度，分 foundation / data / llm / eval 四类。
- 每个技能带 `learnability`（可学习性）、`min_level`（岗位要求水位）、`weight`（权重）、`branch_impact`（分支影响）。
- 依赖图（`dependencies`）支持跨维度先修关系，用于推导「下一步最值得学什么」的拓扑约束。
- 支持 orientation（如 RAG 检索向）对权重 / 目标水位做增量调整，而非另起一套图谱。

## 3. RAG 资源推荐

- `text-embedding-3-small` 向量化 + pgvector / HNSW 语义召回。
- **多信号重排**：相关性 + 时效性 + 岗位适配度，多信号加权重排。
- 离线评测指标：NDCG@10 / Hit@3 / Recall@5（mock embedder，无需库 / 网络）。

## 4. 资源策展 Agent

- Function Calling 编排「搜索 → 抓取 → 摘要 → 入库 → 保鲜校验」工具链。
- 按 `url_hash` 幂等去重，重复 URL 不重复入库。
- 定期检测来源时效（freshness），过期资源降权 / 标注。

## 5. 零输入前端

- 技能胶囊选择 + React Flow 职业能力图谱，全程无需打字。
- 三态可视化（已具备 / 部分 / 缺口）直接呈现能力差距。

## 6. 工程基线

- 数据库 schema 唯一事实源是 Alembic 迁移，应用启动不 `create_all`，只做技能引用完整性校验（fail-fast）。
- 全栈双语 i18n：后端按 `Accept-Language` 返回（默认 en，zh 可切），前端 next-intl + cookie 定位；决策层语言中立，仅表达层本地化。
