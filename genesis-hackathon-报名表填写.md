**5. 项目详细介绍 Project Description**（300–800 字）✱

```
Zeno 是一款面向开发者与产品经理的 AI 职业迁移导航。

【解决什么问题】技术转型最难的从来不是缺学习资料，而是判断——我已有的哪些能力能迁移到 AI 岗位？我真正缺的是什么？众多要补的技能里该先学哪个？大多数工具要么恭维用户、要么把人淹没。Zeno 只诚实地做一件事：用真实招聘数据告诉你，你站在哪、哪些能力能带走、最有效的下一步是什么。

【目标用户】想转型 AI 的前端 / 后端 / 全栈工程师与学生，以及希望转 AI PM 的产品经理。

【核心功能】(1) 交互式"能力星图"点亮已可迁移的工程基础，能力链从 TypeScript→API→流式→Prompt→工具调用→Agent 一路长出；(2) 随滚动推进的"迁移地图"，给出可迁移优势、真实差距（核心/加分）以及按优先级排序、配好学习资源的路线图；(3) 贴一段真实招聘 JD，自动识别其偏向 RAG / Agent / 评估，并据此重新校准差距与路线。支持 English / 简体中文 / 繁體中文。

【技术方案】Zeno 把"决策"与"表达"分开：确定性决策引擎跑在一张用真实招聘数据校准的 23 技能图谱上，计算优势、差距、技能依赖与优先级，结果可复现、有据可依；LLM（DeepSeek 或任意兼容 OpenAI 的服务）只负责把诊断翻译成自然语言，绝不参与判断。学习资源由一个小型 RAG 引擎推荐：基于 pgvector 的向量召回 + 多信号重排（相关度、时效、等级匹配）。前端 Next.js 15，后端 FastAPI，数据库 Postgres 16 + pgvector。

【AI×Web3 融合】评估完成后，前端调用 Transfer Passport 合约，把用户的迁移旅程铸成一枚 Soulbound（ERC-5192，不可转让）SBT：起点角色 → 目标角色 + 迁移就绪度 / 可迁移优势 / 待解锁缺口。图与元数据完全链上现场渲染 SVG，无需 IPFS 或后端。这样你的成长与迁移记录属于你、可携带到任何 AI 学习或招聘平台，而不锁在某个平台的数据库里——Web2 证明"结果"，Web3 让你拥有并证明"迁移过程"。合约为标准 EVM，已可部署到 Base Sepolia。
```
