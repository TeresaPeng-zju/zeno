# Zeno · AI³（0G 赛道）作品描述 — 加详版

> 直接复制粘贴进「作品描述」字段。

```
Zeno 是一款面向开发者与产品经理的 AI 职业迁移导航：用真实招聘数据告诉你，哪些能力能迁移到 AI 岗位、真正缺什么、该先补哪个。

【解决什么问题】技术转型最难的不是缺学习资料，而是判断——我已有的能力哪些能迁移？真正缺什么？该先学哪个？大多数工具要么恭维用户、要么把人淹没；而且 AI 给的诊断往往是黑箱，你无从验证它是不是认真算的。

【目标用户】想转型 AI 的前端 / 后端 / 全栈工程师与学生，以及希望转 AI PM 的产品经理。

【核心设计：决策与表达分离】一个确定性引擎跑在用真实招聘数据校准的 23 技能图谱上，计算优势、差距、技能依赖与优先级——结果可复现、有据可依；LLM 只负责把诊断翻译成自然语言，绝不参与判断。

【0G 集成（本赛道核心）】正因为表达层被干净地隔离出来，我们把它整体接到 0G Private Computer：
- 通过 0G 官方 OpenAI 兼容 Router（endpoint: router-api…/v1）调用去中心化模型（gpt-oss-120b / 0GM 自研模型），不经任何第三方 AI API。
- 每次生成从响应中取出 0G 的 request id，由 TEE 背书、可链上追溯。
- 结果页实时显示「0G Verifiable Inference · request id」验证徽章。
- 边界清晰：决策（差距/排序/就绪度）在确定性引擎，只有"表达层"走 0G——所以既可复现、又可验证。
这正是「可验证推理」的价值落地：AI 给的职业建议不再是黑箱，每一句诊断都跑在去中心化、可验证的算力上，可追溯、不被单一平台垄断或篡改。

【Web3 场景】评估后可一键生成一枚 Soulbound 迁移护照（ERC-5192 不可转让 SBT，链上现场渲染 SVG），把用户的迁移旅程变成可携带、归自己所有的凭证。

【技术栈】前端 Next.js 15 + React Flow；后端 FastAPI；数据库 Postgres 16 + pgvector；RAG 资源召回（向量召回 + 多信号重排）；表达层推理由 0G Compute 提供。开源于 Apache License 2.0（Open Core，自带示例技能图谱，开箱即用）。

【赛题自检】

✓ 结合 Web3 场景（链上 Soulbound 护照）
```
